Interactive UR5e Forward & Inverse Kinematics

Understanding the relationship between a robot's joint angles and its end-effector position/orientation is fundamental in robotics. This interactive tool demonstrates Forward Kinematics (FK) and Inverse Kinematics (IK) for the Universal Robots UR5e arm.

How it Works

  • Forward Kinematics (FK): Given a set of joint angles $q = (q_1, q_2, q_3, q_4, q_5, q_6)$, FK calculates the resulting pose (position and orientation) of the robot's Tool Center Point (TCP). Use the sliders below to manipulate the joint angles (in radians). The solid red robot arm shows the configuration resulting from your chosen angles. The larger coordinate frame indicates the calculated TCP pose.
  • Inverse Kinematics (IK): Given a desired TCP pose $T_{desired}$, IK calculates the set(s) of joint angles $q$ that can achieve that pose. For a 6-DOF arm like the UR5e, there can be multiple solutions (typically up to 8). This tool takes the TCP pose calculated from the FK (based on your slider inputs) and feeds it into the IK solver. The dashed, colored robot arms represent all the valid IK solutions found for that target TCP pose.

Interact!

Adjust the sliders for each joint ($J_1$ to $J_6$) to see how the robot moves (FK). Observe how the different possible IK solutions (dashed lines) change as the target TCP pose moves. Note that some configurations might be unreachable or singular, resulting in fewer or no IK solutions.

Implementation Details

The visualization uses Three.js for 3D rendering. The core FK and IK calculations are based on the standard Denavit-Hartenberg (DH) parameters for the UR5e and a geometric IK solution approach, translated from Python to JavaScript.

The IK solver attempts to find all possible joint configurations that reach the target TCP pose derived from the FK sliders. Due to numerical precision and potential singularities, the number of solutions found might vary.
